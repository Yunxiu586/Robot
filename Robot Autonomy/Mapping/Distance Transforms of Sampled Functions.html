Introduction

Let G be a regular grid and let f:G→R be a sampled function. The distance transform of f is defined as

Df(p)=minq∈G(d(p,q)+f(q))

For each grid point p, the transform searches for a point q that has both a small value f(q) and a small spatial cost d(p,q). Thus, the output function combines local cost information with distance-based structure.

Minimum Convolution

The transform is closely related to minimum convolution. For two functions f and g, the min-convolution is

(f⊗g)(p)=minq(f(q)+g(p−q))

If the distance has the translation-invariant form

d(p,q)=g(p−q)

then the distance transform becomes

Df(p)=(f⊗g)(p)
Classical EDT

The classical Euclidean distance transform of a binary image is obtained as a special case. Let

f(q)={0q∈P+∞q∉P

where P is the set of feature points. Applying the squared Euclidean transform gives

Df(p)=minq∈P∥p−q∥2

Taking the square root gives the exact Euclidean distance to the nearest feature point.

Squared Euclidean distance of one dimension

Problem Definition

For a one-dimensional grid G={0,…,n−1}, the squared Euclidean distance transform is

Df(p)=minq∈G((p−q)2+f(q))

A direct computation would evaluate every pair (p,q), producing an O(n2) algorithm.

For each fixed q, the expression

(p−q)2+f(q)

is a parabola as a function of p. Hence the distance transform is the lower envelope of n parabolas. The value Df(p) is the height of this lower envelope at location p.

Consider two parabolas generated by grid positions q and r:

(p−q)2+f(q)
(p−r)2+f(r)

Their intersection point satisfies

(p−q)2+f(q)=(p−r)2+f(r)

Solving for the horizontal coordinate gives

s=(f(r)+r2)−(f(q)+q2)2r−2q

If q<r, then the parabola from q lies below the parabola from r to the left of s, and above it to the right of s.

Lower Envelope Algorithm

Algorithm DT(f)
    k <- 0
    v[0] <- 0
    z[0] <- -infinity
    z[1] <- +infinity


    for q = 1 to n - 1 do
        s <- intersection(q, v[k])
        while s <= z[k] do
            k <- k - 1
            s <- intersection(q, v[k])
        k <- k + 1
        v[k] <- q
        z[k] <- s
        z[k + 1] <- +infinity


    k <- 0
    for q = 0 to n - 1 do
        while z[k + 1] < q do
            k <- k + 1
        D_f[q] <- (q - v[k])^2 + f[v[k]]

The lower envelope is represented using two arrays and an index.

  • v[i] is the grid location of the i-th parabola in the lower envelope.
  • z[i] is the boundary location at which the i-th parabola begins to be lower than the previous one.
  • k is the index of the rightmost parabola currently in the lower envelope.

In other words, the parabola from v[i] is the active lower-envelope parabola on an interval whose boundaries are given by z[i] and z[i+1].

The algorithm processes the parabolas from left to right according to the horizontal locations of their vertices. When a new parabola from location q is considered, the algorithm computes its intersection with the rightmost parabola currently in the envelope, namely the parabola from v[k].

The first loop constructs the lower envelope.

If the intersection point s satisfies

s>z[k],

then the existing rightmost parabola remains active from z[k] to s, and the new parabola becomes active after s. The new parabola is therefore inserted into the lower envelope.

If

s≤z[k],

then the new parabola is already below the current rightmost parabola throughout the latter's supposed active interval. Consequently, the current rightmost parabola cannot be part of the lower envelope and is removed. The algorithm then tests the new parabola against the previous parabola in the envelope.

This deletion process can repeat, but each parabola can be inserted at most once and removed at most once. The lower-envelope construction therefore takes O(n).

The second loop evaluates the transform.

After the envelope is constructed, the algorithm scans the grid locations q from left to right. For each position q, it finds the active parabola and evaluates

Df(q)=(q−v[k])2+f(v[k])

The scan over the grid and the scan over the active parabolas are both monotone. Hence the sampling step also takes O(n).

The full one-dimensional squared Euclidean distance transform is therefore computed in O(n) time.

Squared Euclidean distance of arbitrary dimensions

The squared Euclidean distance is separable across dimensions. In two dimensions,

Df(x,y)=minx′,y′((x−x′)2+(y−y′)2+f(x′,y′))

This can be decomposed as

Df(x,y)=minx′((x−x′)2+miny′((y−y′)2+f(x′,y′)))

The inner minimization is a one-dimensional transform along one dimension, and the outer minimization is another one-dimensional transform along the other dimension.

Thus, a two-dimensional transform can be computed by first transforming all columns and then transforming all rows. The same principle extends to arbitrary dimensions.

For a d-dimensional grid with N sample points, the total running time is

O(dN)

This separability is essential for practical Euclidean distance transform algorithms in images, volumes, and voxel grids.

L1 Distance Transform

In one dimension, the transform is

Df(p)=minq(|p−q|+f(q))

This transform can be computed by two passes over the array. The forward pass propagates information from left to right

Df(q)=min(Df(q),Df(q−1)+1)

The backward pass propagates information from right to left

Df(q)=min(Df(q),Df(q+1)+1)

Together, these passes compute the exact one-dimensional L1 distance transform in O(n) time. Since the multi-dimensional L1 distance is also separable, the same dimension-by-dimension strategy gives an O(dN) algorithm.

Algorithm DT(f)
    D_f <- f


    for q = 1 to n - 1 do
        D_f[q] <- min(D_f[q], D_f[q - 1] + 1)


    for q = n - 2 down to 0 do
        D_f[q] <- min(D_f[q], D_f[q + 1] + 1)

The initialization Df←f means that each position first keeps its own original cost.

The forward pass considers sources located at or to the left of the current position. If the best value at q−1 is already known, then reaching q from that source adds one unit of L1 distance, giving Df(q−1)+1.

The backward pass performs the symmetric operation. It considers sources located at or to the right of the current position. If the best value at q+1 is known, then reaching q adds one unit of distance, giving Df(q+1)+1.

After both passes, each position has received the best possible contribution from both directions. Therefore, the final value satisfies

Df(p)=minq(f(q)+|p−q|)

The algorithm performs a constant amount of work per grid point in each pass, so its running time is O(n).
