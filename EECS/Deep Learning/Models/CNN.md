# CNN

[toc]

A **Convolutional Neural Network** is a neural network architecture designed for grid-like data, such as images. 
Compared with fully connected layers, convolutional layers introduce two important inductive biases

- translation equivariance through weight sharing
- locality

These structural assumptions greatly reduce the number of learnable parameters and allow the network to extract spatial features efficiently.

### Convolutional Layer

A convolutional layer can be viewed as a structured fully connected layer with local connectivity and parameter sharing.

Let the input be $X$ and the output be $H$. For a general fully connected layer, each output element depends on all input elements

$$
h_{i,j}
=
\sum_{k,l}
w_{i,j,k,l}x_{k,l}
$$

By re-indexing the input position relative to the output position, we write

$$
h_{i,j}
=
\sum_{a,b}
v_{i,j,a,b}x_{i+a,j+b}
$$

where $i,j$ specify the output position, and $a,b$ specify the relative spatial offset.

##### Translation Equivariance

Translation equivariance is obtained by using the same feature detector at different spatial positions. Therefore, the parameter no longer depends on the output location

$$
v_{i,j,a,b}
=
v_{a,b}
$$

Then the output becomes

$$
h_{i,j}
=
\sum_{a,b}
v_{a,b}x_{i+a,j+b}
$$

This operation is usually called **2D convolution** in deep learning, although it is mathematically closer to **cross-correlation**.

##### Locality

Locality assumes that an output element only depends on a local neighborhood of the input.

When the relative offset is too large, the weight is set to zero

$$
v_{a,b}
=
0
\qquad
|a|>\Delta
\ \text{or}\
|b|>\Delta
$$

Thus, the convolution operation becomes

$$
h_{i,j}
=
\sum_{a=-\Delta}^{\Delta}
\sum_{b=-\Delta}^{\Delta}
v_{a,b}x_{i+a,j+b}
$$

The learnable parameters $v_{a,b}$ form a convolution kernel or filter with size

$$
(2\Delta+1)\times(2\Delta+1)
$$

The kernel slides over the input and computes local dot products to extract spatial features.

##### 2D Convolution

For a single-channel 2D convolution layer, let

$$
X\in\mathbb{R}^{n_h\times n_w}
$$

$$
W\in\mathbb{R}^{k_h\times k_w}
$$

$$
b\in\mathbb{R}
$$

Then the output is

$$
Y
=
X*W+b
$$

where $W$ and $b$ are learnable parameters.

The output shape is

$$
(n_h-k_h+1)\times(n_w-k_w+1)
$$

The kernel size $k_h\times k_w$ is a hyperparameter.

##### Padding

Padding adds extra rows and columns around the input. If $p_h$ rows are padded on each side in height and $p_w$ columns are padded on each side in width, the output shape becomes

$$
(n_h+2p_h-k_h+1)
\times
(n_w+2p_w-k_w+1)
$$

Padding is commonly used to control the spatial size of the output.

To keep the input and output shapes unchanged, when $k_h$ and $k_w$ are odd, a common choice is

$$
p_h=\frac{k_h-1}{2}
$$

$$
p_w=\frac{k_w-1}{2}
$$

##### Stride

Stride specifies the step size of the sliding kernel.

Let the stride be $s_h$ in height and $s_w$ in width. With per-side padding $p_h$ and $p_w$, the output shape is

$$
\left(
\left\lfloor
\frac{n_h+2p_h-k_h}{s_h}
\right\rfloor+1
\right)
\times
\left(
\left\lfloor
\frac{n_w+2p_w-k_w}{s_w}
\right\rfloor+1
\right)
$$

A larger stride reduces the output spatial size more aggressively.

##### Multiple Input Channels

For one output channel, each input channel is associated with one 2D kernel.

Let

$$
X\in\mathbb{R}^{c_i\times n_h\times n_w}
$$

$$
W\in\mathbb{R}^{c_i\times k_h\times k_w}
$$

$$
b\in\mathbb{R}
$$

The output is obtained by summing the convolution results over all input channels:

$$
Y
=
\sum_{i=1}^{c_i}
X_{i,:,:}*W_{i,:,:}
+
b
$$

The output has one channel.

##### Multiple Output Channels

To produce multiple output channels, the layer uses multiple 3D kernels.

Let

$$
X\in\mathbb{R}^{c_i\times n_h\times n_w}
$$

$$
W\in\mathbb{R}^{c_o\times c_i\times k_h\times k_w}
$$

$$
B\in\mathbb{R}^{c_o}
$$

Then the output is

$$
Y\in\mathbb{R}^{c_o\times m_h\times m_w}
$$

Each output channel is generated by one kernel:

$$
Y_{o,:,:}
=
\sum_{i=1}^{c_i}
X_{i,:,:}*W_{o,i,:,:}
+
B_o
\qquad
o=1,2,\dots,c_o
$$

The convolution kernel is therefore a 4-dimensional learnable tensor:

$$
W
\in
\mathbb{R}^{c_o\times c_i\times k_h\times k_w}
$$

where

$$
c_o
=
\text{out channels}
$$

$$
c_i
=
\text{in channels}
$$

$$
k_h
=
\text{kernel height}
$$

$$
k_w
=
\text{kernel width}
$$

##### $1\times1$ Convolution

A $1\times1$ convolution uses

$$
k_h=k_w=1
$$

It does not capture spatial patterns directly. Instead, it mixes information across channels at each spatial location.

For an input with $c_i$ channels and an output with $c_o$ channels, a $1\times1$ convolution is equivalent to applying a fully connected layer with weight shape

$$
c_o\times c_i
$$

independently at each spatial position.

##### General Form of 2D Convolution

For a general 2D convolution layer, let

$$
X\in\mathbb{R}^{c_i\times n_h\times n_w}
$$

$$
W\in\mathbb{R}^{c_o\times c_i\times k_h\times k_w}
$$

$$
B\in\mathbb{R}^{c_o}
$$

$$
Y\in\mathbb{R}^{c_o\times m_h\times m_w}
$$

Then

$$
Y
=
X*W+B
$$

where $B$ is broadcast over the spatial dimensions.

The computational complexity is

$$
O(c_i c_o k_h k_w m_h m_w)
$$

### Pooling Layer

A pooling layer reduces the sensitivity of convolutional features to small spatial shifts.

Pooling layers have no learnable parameters. Their main hyperparameters are

- window size
- padding
- stride

Pooling is applied independently to each input channel. Therefore, the number of output channels is the same as the number of input channels.

##### Max Pooling

Max pooling keeps the strongest activation inside each local window:

$$
\operatorname{Output}
=
\max(\operatorname{Window})
$$

It is useful for detecting whether a strong local feature exists.

##### Average Pooling

Average pooling computes the mean activation inside each local window:

$$
\operatorname{Output}
=
\operatorname{mean}(\operatorname{Window})
$$

It summarizes the average response of a local region.

### Fully Connected Layer

A fully connected layer is also called a dense layer or a linear layer.

Each input neuron is connected to each output neuron. For an input vector $\boldsymbol{\mathit{x}}$, the layer computes
$$
\boldsymbol{\mathit{y}}
=
W\boldsymbol{\mathit{x}}
+
\boldsymbol{\mathit{b}}
$$

where $W$ is a learnable weight matrix and $\boldsymbol{\mathit{b}}$ is a learnable bias vector.
